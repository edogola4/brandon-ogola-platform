# Contributing

Please read GOVERNANCE.md and CODE_OF_CONDUCT.md. Follow these guidelines:

- Branching: use feature/*, fix/*, chore/* prefixes. Create PRs against main.
- Commit messages: follow Conventional Commits (type(scope): subject)
- PR checklist: include tests, update docs, link related issues
- Definition of Done: code reviewed, CI passing, tests added for new behaviour
